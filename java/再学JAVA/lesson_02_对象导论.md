1. 抽象

"人们所能解决的问题的复杂性直接取决与抽象的类型和质量". 怎么理解呢?

计算机    -- 手工编程-----------------------------------------------------   人类
计算机    ---------- 机器语言---------------------------------------------   人类
计算机    --------------------汇编语言------------------------------------   人类
计算机    ----------------------------Fortran/BASIC/C--------------------   人类
计算机    --------------------------------------------C++/Java/Python----   人类

每一层的抽象都让人编程更容易了, 能解决的问题也更复杂了.

2. 对象

"每个对象都提供服务". 怎么理解呢?
看<Java编程思想>, 这句话印象最深.

写代码时, 多思考这个对象对外提供什么服务. 只把对外的开放出来, 而其他的核心内在的保护好, 隐藏好. 这就是封装.
例如: 人最核心的是大脑/心脏. 大脑/心脏并没有直接暴露出来, 而是有头骨和肋骨保护得好好的.
